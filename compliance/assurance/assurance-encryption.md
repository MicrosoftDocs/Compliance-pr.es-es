<span data-ttu-id="cd936-118">Los productos y servicios de Microsoft 365 usan protocolos de transporte fuertes, como TLS, para evitar que partes no autorizadas intercepten datos de clientes mientras se mueven a través de una red.</span><span class="sxs-lookup"><span data-stu-id="cd936-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="cd936-119">Algunos ejemplos de datos en tránsito son los mensajes de correo que están en proceso de entrega, las conversaciones que se realizan en una reunión en línea o los archivos que se replican entre los centros de datos.</span><span class="sxs-lookup"><span data-stu-id="cd936-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="cd936-122">¿Cómo administra Microsoft 365 las claves usadas para el cifrado?</span><span class="sxs-lookup"><span data-stu-id="cd936-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="cd936-130">Los clientes con requisitos para controlar sus propias claves de cifrado raíz pueden aprovechar el cifrado de servicio con la clave de cliente.</span><span class="sxs-lookup"><span data-stu-id="cd936-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="cd936-131">Mediante el uso de la clave de cliente, los clientes pueden generar sus propias claves criptográficas mediante un módulo de servicio de hardware (HSM) local o Azure Key Vault (AKV).</span><span class="sxs-lookup"><span data-stu-id="cd936-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="cd936-132">Las claves raíz del cliente se almacenan en AKV, donde se pueden usar como la raíz de una de las llaves que cifran los archivos o datos de buzones de correo del cliente.</span><span class="sxs-lookup"><span data-stu-id="cd936-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="cd936-133">El código de servicio 365 de Microsoft para el cifrado de datos solo puede tener acceso directo a las claves raíz del cliente, y los empleados de Microsoft no pueden acceder a ellas.</span><span class="sxs-lookup"><span data-stu-id="cd936-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
