# <a name="supplier-management-overview"></a>Información general de administración de proveedores

## <a name="how-does-microsoft-manage-risk-related-to-suppliers"></a>¿Cómo administra Microsoft los riesgos relacionados con los proveedores?

Microsoft se asocia con compañías de terceros para ayudar a satisfacer las necesidades de nuestros clientes. Estas compañías de terceros se denominan proveedores o subprocesadores. La seguridad y la privacidad de los proveedores en Microsoft se rige por nuestro [programa Supplier Security and Privacy Assurance (SSPA)](https://www.microsoft.com/procurement/sspa?activetab=pivot1%3aprimaryr6), un conjunto de requisitos de toda la empresa para todos los proveedores que se asocian con Microsoft para ofrecer nuestros servicios en línea. Aunque el programa de SSPA proporciona un control y una administración completos de nuestra base de proveedores, las unidades de negocio individuales, como Microsoft 365, pueden mantener requisitos adicionales para sus proveedores.

## <a name="how-does-microsofts-supplier-security-and-privacy-assurance-sspa-program-protect-customer-data"></a>¿Cómo protege el programa la seguridad del proveedor y la garantía de privacidad de Microsoft (SSPA) los datos de los clientes?

SSPA es una sociedad entre la adquisición de Microsoft, las asuntos legales y externas de la empresa, y la seguridad corporativa para garantizar que los proveedores sigan los principios de privacidad y seguridad de Microsoft. El ámbito de la SSPA cubre a todos los proveedores que procesan datos personales o datos confidenciales de Microsoft. La inscripción del programa SSPA incluye el cumplimiento de los requisitos de la protección de datos de Microsoft (DPR). DPR consta de controles de seguridad y privacidad que los proveedores deben implementar antes de comenzar el trabajo contratado con Microsoft. Todos los proveedores inscritos se atestiguan automáticamente para cumplir con el DPR anual.

Los requisitos de DPR se basan en seis categorías de procesamiento de datos distintas que un proveedor puede aprobar como parte de su inscripción en la SSPA. Estas categorías se usan para identificar el riesgo asociado con los servicios que proporciona un proveedor a Microsoft. El perfil de procesamiento de datos del proveedor determina qué controles de DPR se consideran en el ámbito para proporcionar la protección de datos adecuada. Los proveedores que procesan datos que se consideran un riesgo mayor deben cumplir con todos los requisitos de DPR y también pueden necesitar la comprobación independiente del cumplimiento. Las herramientas de compra de Microsoft validan el estado de SSPA de todos los proveedores, incluido el cumplimiento de las partes aplicables del DPR, antes de permitir la adquisición de ese proveedor.

## <a name="what-types-of-subprocessors-provide-services-for-microsoft"></a>¿Qué tipos de subprocesadores proporcionan servicios para Microsoft?

Un ' subprocesador ' es un tercero que contrata Microsoft cuyas obligaciones incluyen el procesamiento de datos personales de Microsoft para los que Microsoft es un procesador. Los subprocesadores de Microsoft se dividen en tres categorías distintas. Cada uno debe demostrar el cumplimiento de la SSPA antes de que pueda procesar los datos del cliente en nombre de Microsoft.

- Los subprocesadores de **tecnología** proporcionan tecnologías que se usan para proporcionar servicios de Microsoft online específicos. Si un cliente implementa uno de estos servicios, los subprocesadores identificados para ese servicio pueden procesar, almacenar o tener acceso a los datos personales o datos de los clientes mientras ayudan a proporcionar dicho servicio.
- Los subprocesadores **auxiliares** proporcionan servicios que admiten, operan y mantienen los servicios en línea. Si un cliente implementa uno de estos servicios, los subprocesadores identificados pueden procesar, almacenar o tener acceso a datos de clientes o datos personales limitados, a la vez que proporcionan sus servicios auxiliares.
- Los subprocesadores de **aumento de personal** tienen dos formas distintas: en ambos escenarios, los datos personales residen solo en las instalaciones de Microsoft, en sistemas Microsoft y están sujetos a la supervisión y las directivas de Microsoft.

    - La primera forma de aumentar el personal proporciona personal que admite, opera y mantiene los servicios en línea de Microsoft. A la vez que se cumplen sus responsabilidades, estos subprocesadors pueden estar expuestos a datos de clientes o datos personales. Por ejemplo, un subprocesador puede solucionar problemas de forma remota en un servidor de Microsoft y, al hacerlo, se puede exponer a fragmentos de datos de clientes en un registro de volcado de bloqueos del servidor.
    - La segunda forma de aumento del personal implica a subprocesadores que trabajan en paralelo con los empleados de tiempo completo de Microsoft para apoyar, operar y mantener los servicios en línea de Microsoft. Estos subprocesadores pueden estar expuestos a datos de pseudonymized como parte de su trabajo junto con los empleados de tiempo completo de Microsoft.

La tecnología y los subprocesadors auxiliares son necesarios para implementar controles de acceso conforme a los requisitos de la protección de datos de Microsoft (DPR). Estos requisitos cubren o superan los compromisos contractuales que Microsoft hace a sus clientes en los términos del servicio en línea (OST). Los proveedores que realizan trabajo de aumento del personal están sujetos a los mismos controles de acceso en su lugar para los empleados de tiempo completo de Microsoft.

## <a name="how-does-microsoft-onboard-suppliers"></a>¿Cómo incorpora Microsoft los proveedores?

Se requieren proveedores de terceros para firmar un acuerdo de Microsoft Master como parte del proceso de incorporación. Este Contrato rige la relación entre Microsoft y sus proveedores y garantiza una administración coherente de las relaciones con los proveedores. Como parte de la incorporación, los proveedores se inscriben en la SSPA y deben completar todos los requisitos aplicables antes de que se puedan aprobar para las categorías de procesamiento de datos. Las unidades de negocio de Microsoft solo pueden crear negociaciones con los proveedores cuando la actividad de procesamiento de datos de la contratación coincide con las categorías de procesamiento de datos para las que se ha aprobado el proveedor.

## <a name="how-does-microsoft-notify-customers-of-changes-to-suppliers-who-process-their-data"></a>¿Cómo notifica Microsoft a los clientes los cambios a los proveedores que procesan sus datos?

Según el anexo de protección de datos de servicios en línea de Microsoft (DPA), Microsoft realiza compromisos adicionales con respecto a los períodos de aviso para la adición de cualquier subprocesador. Los intervalos de tiempo de los avisos dependen del tipo de datos que el subproceso procesará en nombre de Microsoft. Como se indicó en DPA, Microsoft se compromete a ofrecer avisos a los clientes al menos seis meses antes que cualquier nuevo subprocesador que procese los datos de los clientes. Para cualquier otro dato personal, Microsoft proporcionará al menos 30 días de antelación. La actualización de la [lista de subprocesadores de Microsoft Online Services](https://servicetrust.microsoft.com/ViewPage/TrustDocumentsV3?command=Download&downloadType=Document&downloadId=926b2cf5-6b6e-43ca-9bc3-f73e961aad5f&tab=7f51cb60-3d6c-11e9-b2af-7bb9f5d2d913&docTab=7f51cb60-3d6c-11e9-b2af-7bb9f5d2d913_Subprocessor_List)proporciona un aviso.
